# Landing Page Deploy

Get your landing page live in 10 minutes. Pick Vercel or Netlify (both free tier). Add a custom domain if you want (optional).

---

## Quick Start

### Vercel (Recommended)

Vercel is made by the Next.js team. Works with Astro, plain HTML, and Node.js backends.

**1. Install Vercel CLI**

```bash
npm i -g vercel
```

**2. Deploy**

```bash
vercel
# Follow prompts. It will ask:
# - Connect your GitHub account? (optional, but easier for auto-deploy)
# - Which folder? (. for current, or dist/)
# - Project name?
```

**3. That's it.**

Live at: `your-project.vercel.app`

### Netlify (Also good)

Alternative if you prefer Netlify's UI.

**1. Install Netlify CLI**

```bash
npm i -g netlify-cli
```

**2. Deploy**

```bash
netlify deploy --prod
# Follow prompts. It will ask:
# - Publish folder? (. or dist/)
# - Team? (skip if first time)
```

**3. Live at:** `your-project.netlify.app`

## Deploy Plain HTML

### Step 1: Build Tailwind

```bash
npm run build
```

This creates `dist/style.css`.

### Step 2: Deploy

**Vercel:**

```bash
vercel --prod
```

At the "Publish folder" prompt, enter: `src` (or wherever your index.html is).

**Netlify:**

```bash
netlify deploy --prod --dir=src
```

### Folder structure for deploy

```
landing-page/
├── src/
│   ├── index.html
│   ├── script.js
│   └── dist/
│       └── style.css (generated by Tailwind)
├── public/
│   └── images/
```

When Vercel asks "Publish folder?" -- choose `src`.

## Custom Domain Setup

### On Vercel

1. Go to vercel.com/dashboard
2. Select your project
3. Settings > Domains
4. Add your domain (e.g., waitlist.yourcompany.com)
5. Vercel shows DNS instructions
6. Go to your domain registrar (GoDaddy, Namecheap, etc)
7. Add the CNAME record that Vercel gives you
8. Wait 10 minutes for DNS to propagate

### On Netlify

1. Go to netlify.com/dashboard
2. Select your site
3. Domain settings > Add custom domain
4. Enter your domain
5. Netlify shows DNS instructions
6. Same as Vercel: go to your registrar, add CNAME record
7. Wait for propagation

### Verify it worked

```bash
# Wait 10 minutes, then:
ping yourdomain.com
# Should resolve to your Vercel/Netlify IP
```

## Environment Variables

Store secrets in environment variables. Never commit them.

### Plain HTML / Astro

Create `.env.local`:

```
STRIPE_PUBLIC_KEY=pk_test_your_key
STRIPE_SECRET_KEY=sk_test_your_key
RESEND_API_KEY=your_key_here
```

**Never commit `.env.local`.** Add to `.gitignore`:

```
.env.local
node_modules/
dist/
```

### On Vercel

1. Go to vercel.com/dashboard
2. Select your project
3. Settings > Environment Variables
4. Add each variable:
   - Name: `STRIPE_PUBLIC_KEY`
   - Value: `pk_test_...`
   - Environments: Check "Production"
5. Save
6. Re-deploy: `vercel --prod`

### On Netlify

1. Go to netlify.com/dashboard
2. Site settings > Build & deploy > Environment
3. Add variables
4. Re-deploy: `netlify deploy --prod`

## Stripe Webhook Setup (Payment confirmation)

Only do this if you have payment on your landing page. Skip for waitlist tests.

### Create a webhook

1. Go to [stripe.com/dashboard](https://stripe.com/dashboard)
2. Developers > Webhooks
3. Add endpoint
4. URL: `https://yourdomain.com/api/stripe-webhook`
5. Events to send: `checkout.session.completed`
6. Save. Copy the signing secret.

### Store the signing secret

Add to Vercel/Netlify environment variables:

```
STRIPE_WEBHOOK_SECRET=whsec_test_...
```

### Create webhook handler

**For Astro, create `src/pages/api/stripe-webhook.ts`:**

```typescript
import type { APIRoute } from 'astro';
import Stripe from 'stripe';

const stripe = new Stripe(import.meta.env.STRIPE_SECRET_KEY);
const webhookSecret = import.meta.env.STRIPE_WEBHOOK_SECRET;

export const POST: APIRoute = async ({ request }) => {
  const signature = request.headers.get('stripe-signature');
  const body = await request.text();

  try {
    const event = stripe.webhooks.constructEvent(
      body,
      signature,
      webhookSecret
    );

    if (event.type === 'checkout.session.completed') {
      const session = event.data.object;
      console.log('Payment confirmed for:', session.customer_email);
      // TODO: Mark user as paid in your database
    }

    return new Response(JSON.stringify({ received: true }), { status: 200 });
  } catch (err) {
    console.error('Webhook error:', err);
    return new Response('Webhook error', { status: 400 });
  }
};
```

**For plain HTML, use a serverless function or skip for now.**

### Test webhook locally

```bash
stripe listen --forward-to localhost:3000/api/stripe-webhook
```

This will show you webhook events as they happen.

## Health Check: Is deployment working?

### Check 1: Site loads

Open `https://yourdomain.com` in a browser. You should see your landing page.

### Check 2: Email form works

Fill in the waitlist form. Submit. Check your email inbox for confirmation.

**If no email arrives:**
- Check spam folder
- Check your email service (Resend, Mailchimp) logs
- Check browser console for errors (F12 > Console)
- Check Vercel/Netlify logs: `vercel logs` or via dashboard

### Check 3: Images load

Right-click on an image. Inspect. Check the img src. Should start with `https://yourdomain.com` or a CDN URL.

**If images are broken:**
- Make sure you added `public/images/` to your deploy
- Check that image filenames match in HTML
- For Astro, images should be in `public/` not `src/`

### Check 4: Performance

Open your site in Chrome. Press F12 > Lighthouse. Run performance audit.

Target: 90+ performance score. If lower:
- Images too large (use 200KB max for hero)
- Too many scripts (remove unnecessary ones)
- Slow API calls (check Resend/Stripe latency)

### Check 5: Forms submit quickly

Fill form and watch the Network tab (F12 > Network). The POST to `/api/waitlist` should complete in < 1 second.

**If slow:**
- Check if Resend API is responding
- Check if your function has cold start (Vercel/Netlify free tier can be slow first time)
- Add a loading spinner to hide the delay

### Check 6: Stripe test payment (if applicable)

Use Stripe test card: `4242 4242 4242 4242` (any future expiry, any CVC).

- Fill form
- Enter test card
- Complete checkout
- Check Stripe dashboard that payment was created

## Common Deploy Failures

### "Command failed: npm run build"

**Cause:** Missing environment variables.

**Fix:**
```bash
# Locally:
echo STRIPE_PUBLIC_KEY=pk_test_... > .env.local
npm run build
```

Then add those same variables to Vercel/Netlify.

### "Module not found: @stripe/stripe-js"

**Cause:** Dependencies not installed.

**Fix:**
```bash
npm install
vercel --prod
```

### "CORS error on form submission"

**Cause:** Front-end calling `/api/waitlist` but the API doesn't exist or is on a different domain.

**Fix:**
- Make sure your API route file exists: `src/pages/api/waitlist.ts` (Astro) or `api/waitlist.js` (Node.js)
- Make sure you're calling the right path: `/api/waitlist` not `http://yourdomain.com/api/waitlist`
- Check browser console (F12) for actual error message

### "Form submits but email never arrives"

**Cause:** Resend or Mailchimp API key is wrong or not set.

**Fix:**
1. Check Vercel environment variables are set
2. Check the API key is correct (starts with `re_test_` or `re_live_`)
3. In `src/pages/api/waitlist.ts`, add logging:

```typescript
console.log('Sending email to:', email);
console.log('API key:', import.meta.env.RESEND_API_KEY?.slice(0, 10) + '...');
```

4. Re-deploy and check logs: `vercel logs`

### "Webhook not firing"

**Cause:** Signing secret mismatch.

**Fix:**
1. Copy the exact signing secret from Stripe dashboard (Developers > Webhooks > Select endpoint)
2. Add to environment variable as `STRIPE_WEBHOOK_SECRET`
3. Test locally with `stripe listen --forward-to localhost:3000/api/stripe-webhook`
4. Check webhook delivery status in Stripe dashboard

### "Custom domain not working"

**Cause:** DNS hasn't propagated yet or CNAME is wrong.

**Fix:**
```bash
# Check DNS:
dig yourdomain.com
# Should show Vercel/Netlify IP

# Wait 10-30 minutes for full propagation
nslookup yourdomain.com
```

## Zero-downtime updates

Once deployed, you can update your site without downtime:

```bash
# Make changes locally
# Test: npm run dev
# Deploy:
vercel --prod
# Done. New version live in 30 seconds.
```

No need to shut down anything. Just deploy.

## Key Rules for Build Skill

1. Build locally first: `npm run build` and test
2. Add environment variables BEFORE deploying
3. Email capture must be tested end-to-end before going live
4. Custom domain is optional. Vercel/Netlify subdomain is fine for validation tests
5. Check Lighthouse score. 90+ is good
6. Monitor webhook delivery if using Stripe
7. Re-deploy takes 30 seconds. No downtime.
